# Обратная связь по ДЗ · трек MBA (навигатор-стиль)

Редактируемый документ обратной связи для студента по проверенному ДЗ.
Один генератор → одинаковый вид в Word и в Google Документах.

## Стиль (брендбук HAIR EXPERT UNIVERSITY · MBA)
- Палитра: графит `#222A36` (шапка, таблица, заголовки), синий `#254371` (акценты,
  баллы, рамки-колауты), сталь `#818A99` (подзаголовки), почти-чёрный `#13161A`,
  фон плашек `#EDF1F6`, фон выделенного пункта `#E6ECF4`.
- Шрифты: Cormorant (заголовки), Raleway (текст) — оба есть в Google Документах.
- Логотип: `assets/logo-mba.png` (официальный `mba/logo/png/main-light.png`, на белом).
- Красный НЕ используем: главную слабость выделяем колаутом (плашка + синяя левая
  рамка), а не цветом текста.

## Структура
Логотип + кикер → заголовок (Cormorant) → подзаголовок «Трек · Студент» →
плашка «Итоговая оценка» → приветствие по имени → «Важный контекст» →
таблица баллов (строка ИТОГО на графите, баллы синим) →
«Что у вас получилось хорошо» → «Над чем предлагаю поработать» (первый пункт —
выделенный колаут = главная слабость) → «Моя итоговая рекомендация» (плашка).

## Как собрать
```bash
node build_feedback_docx.js <data.json> "<out.docx>"   # логотип берётся из assets/ автоматически
```
`docx` (npm) уже установлен. Схема JSON — `feedback_schema.md`,
готовый пример — `example_feedback.json` (проверка ДЗ №1, трек «Преподавание»).

Текст пишем от первого лица, с обращением к студенту по имени; хвалим и критикуем
конкретикой из работы. Баллы берём из эталона ДЗ, а не по памяти.

## Доставка
- `.docx` — студенту в чат.
- Тот же `.docx` — на Google Drive (конвертация в Google Документ, вёрстка
  сохраняется точнее, чем при загрузке HTML). Папка и правило именования файла
  (`Фамилия ДЗN трек`) — в памяти проекта.
